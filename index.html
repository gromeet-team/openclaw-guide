📝 왜 Claude 기준에서 GPT 기준으로 바꿨나?
2026-04-05에 Anthropic billing/usage reject와 Credit balance is too low 이슈가 실제 운영 크론 실패로 이어졌고, 같은 날 코어 운영은 이미 openai-codex/gpt-5.4로 안정 운용 가능하다고 확인됐습니다. 그래서 현재 문서는 GPT-first를 기본으로 정리하고, Claude는 필요 시 fallback/선택 옵션으로만 다룹니다.
WSL2란?
Windows 안에 리눅스를 돌릴 수 있게 해주는 윈도우 기능. 쉽게 말해 "가상 리눅스 컴퓨터"를 내 PC 안에 하나 더 만드는 것.
Ubuntu란?
WSL2 위에서 실제로 실행되는 운영체제(리눅스 종류 중 하나). OpenClaw는 이 Ubuntu 안에 설치됨.
OpenClaw란?
AI 어시스턴트(GPT 중심)를 슬랙·텔레그램 등 채널에 연결하고, 크론잡·자동화를 돌릴 수 있게 해주는 런타임 프레임워크.
GPT OAuth란?
ChatGPT/OpenAI 계정으로 로그인해서 OpenClaw에서 GPT를 쓰는 인증 방식. API 키 없이 구독 계정 기반으로 연결합니다.
💡 전체 소요 시간: 약 30~60분. 처음 한 번만 설치하면 이후 업데이트는 명령어 한 줄이에요.

WSL2 + Ubuntu 설치
윈도우에서 시작
A
PowerShell을 관리자 권한으로 열기

윈도우 검색창에 PowerShell 입력 → 우클릭 → "관리자 권한으로 실행" 클릭

B
WSL2 + Ubuntu 한 번에 설치

아래 명령어를 복사해서 PowerShell에 붙여넣고 Enter

PowerShell (관리자) 
wsl --install -d Ubuntu-24.04
⚠️ 설치 후 컴퓨터 재부팅 요청이 나오면 재부팅하세요. 재부팅 후 Ubuntu가 자동으로 설치 완료 창을 띄웁니다.
C
Ubuntu 계정 설정

재부팅 후 Ubuntu 터미널이 열리면 사용자명과 비밀번호를 설정하라고 나옵니다.

✅ 사용자명: 영문 소문자로 (예: gromit)
✅ 비밀번호: 입력해도 화면에 아무것도 안 표시됩니다 — 정상이에요

D
systemd 활성화 (필수)

Ubuntu 터미널에서 아래 명령어를 차례대로 실행하세요

Ubuntu 터미널 
sudo tee /etc/wsl.conf >/dev/null <<'EOF'
[boot]
systemd=true
EOF

그 다음 PowerShell에서 WSL 재시작:

PowerShell 
wsl --shutdown

Ubuntu를 다시 열고 아래로 확인:

Ubuntu 터미널 
systemctl --user status
✅ 에러 없이 뭔가 출력되면 성공입니다.

3
인증 설정 — GPT OAuth 방식
현재 그로밋팀 기본 운영 경로
🔐 현재 표준은 OpenAI Codex OAuth입니다. 즉, ChatGPT/OpenAI 계정으로 로그인해서 openai-codex/gpt-5.4를 쓰는 방식입니다. API 키와 다르고, 기존 Claude setup-token은 이제 기본 경로가 아닙니다.
📌 운영 정책 요약
① 메인 운영 기준은 GPT-first, ② 결제/좌석은 회사 기준으로 정리, ③ Claude는 필요할 때만 fallback 또는 비교 실험용으로 사용합니다.

4
슬랙 연동
Slack 앱 생성 → OpenClaw 연결
💡 슬랙 연동은 슬랙 워크스페이스 관리자가 해야 합니다. 팀원이 개인용으로 설치하는 경우, 그로밋에게 별도 슬랙 앱 생성을 요청하거나 다른 채널(텔레그램 등)을 사용하세요.
A
슬랙 앱 생성

api.slack.com/apps에서 "Create New App" → "From scratch"

앱 이름 입력 (예: 현비스2) → 워크스페이스 선택 → Create

B
Socket Mode 활성화 + App-Level Token 발급

앱 설정 페이지 → 왼쪽 메뉴 "Socket Mode" → Toggle ON

→ "Generate an app-level token" 클릭
→ 토큰 이름 입력 (예: main)
→ connections:write 스코프 추가
→ Generate

발급된 xapp-1-... 토큰을 복사해두기

C
봇 권한(Scopes) 설정 + Bot Token 발급

왼쪽 메뉴 "OAuth & Permissions" → Bot Token Scopes에 아래 추가:

필수 Scopes
app_mentions:read
channels:history
channels:read
chat:write
groups:history
groups:read
im:history
im:read
im:write
reactions:write
users:read

→ 페이지 위로 스크롤 → "Install to Workspace" 클릭
→ 발급된 xoxb-... Bot Token 복사

D
OpenClaw에 슬랙 토큰 입력
Ubuntu 터미널 
openclaw configure --section channels.slack

또는 ~/.openclaw/openclaw.json에 직접 추가:

openclaw.json — channels 섹션 
{
  "channels": {
    "slack": {
      "mode": "socket",
      "enabled": true,
      "botToken": "xoxb-여기에-Bot-Token-붙여넣기",
      "appToken": "xapp-1-여기에-App-Token-붙여넣기",
      "groupPolicy": "open"
    }
  }
}

저장 후 재시작:

Ubuntu 터미널 
openclaw gateway restart
5
워크스페이스 설정
AI 성격·규칙·메모리 파일 구성

워크스페이스는 AI 어시스턴트의 "두뇌"입니다. 이 파일들이 어시스턴트의 성격, 행동 규칙, 사용자 정보를 정의합니다. 기본 위치: ~/.openclaw/workspace/

SOUL.md
성격/톤 정의 — 어시스턴트가 어떻게 말하고 행동하는지. 캐릭터 설정 파일.
RULES.md
절대 규칙 — 어떤 상황에서도 지켜야 할 행동 원칙. (예: 외부 공유 금지, 삭제 전 확인 등)
USER.md
사용자 정보 — 이름, 직책, 업무 맥락, 선호 등. "내 비서에게 나를 소개하는 파일".
MEMORY.md
장기 메모리 — 중요한 결정사항, 프로젝트 히스토리, 반복되는 맥락 저장.
IDENTITY.md
어시스턴트 정체성 — 이름, 이모지, 캐릭터 설정. (예: HYUNVIS, 🔷)
AGENTS.md
운영 매뉴얼 — 세션 부팅 순서, 메모리 정책, 에이전트팀 운영 규칙 등.
HEARTBEAT.md
헬스체크 체크리스트 — 30분마다 AI가 확인하는 항목들. 이상 있을 때만 DM 발송.
💡 OnBoarding 마법사가 기본 파일들을 자동 생성해줍니다. 그 이후 본인 맥락에 맞게 내용을 채우면 됩니다.

슬랙 운영 가이드
현비스를 슬랙에서 잘 쓰는 법 — 토큰 절약 + 효율 극대화
⚠️ 핵심 규칙: 한 작업 = 한 스레드. 채널에 바로 대화하면 히스토리가 쌓여 토큰이 폭발합니다. 항상 스레드에서 시작하고 스레드에서 끝내세요.

1. 채널 분리 + 스레드 격리 원칙

채널 = 업무 카테고리
채널별로 독립 세션이 생성되어 컨텍스트가 분리됩니다. 업무 종류별로 채널을 나누면 맥락이 섞이지 않아요.
스레드 = 작업 단위
스레드 안의 메시지만 컨텍스트로 들어가므로 토큰이 절약됩니다. 새 작업은 반드시 새 스레드로.

크론잡 (자동화)
정해진 시간에 자동으로 작업을 실행하는 예약 시스템

크론잡(Cron Job)은 정해진 시간에 자동으로 실행되는 예약 작업입니다. 예를 들어 "매일 아침 9시에 뉴스 요약 보내줘", "2시간마다 Google Drive 동기화해줘" 같은 걸 설정하면 사람이 안 시켜도 알아서 실행됩니다.

💡 현비스는 크론잡 20개 이상을 돌리고 있어요 — 인풋 수집, 데일리 공지, 스프린트 체크, AI 뉴스 브리핑 등. OpenClaw의 핵심 가치가 바로 이 자동화입니다.
1
크론잡 목록 확인

현재 등록된 모든 크론잡을 확인합니다.

Ubuntu 터미널 
openclaw cron list

각 항목에 이름, 실행 스케줄, 마지막 실행 상태(ok / error)가 표시됩니다.

2
크론잡 추가하기

새 크론잡을 등록하려면 슬랙 DM에서 어시스턴트에게 자연어로 요청하면 됩니다.

슬랙 DM 예시 
매일 아침 9시에 오늘 일정 요약해서 DM으로 보내줘

어시스턴트가 크론잡을 자동으로 생성합니다. 터미널에서 직접 등록하려면:

Ubuntu 터미널 
# 크론잡 생성 (대화형)
openclaw cron create

# 이름, 스케줄(cron 표현식), 실행할 프롬프트를 입력
3
cron 표현식 이해하기

스케줄은 cron 표현식으로 설정합니다. 5개의 숫자/기호로 "언제 실행할지"를 정의합니다.

cron 표현식 구조 
# ┌─── 분 (0-59)
# │ ┌─── 시 (0-23)
# │ │ ┌─── 일 (1-31)
# │ │ │ ┌─── 월 (1-12)
# │ │ │ │ ┌─── 요일 (0=일, 1=월, ..., 6=토)
# │ │ │ │ │
# * * * * *
표현식의미예시 용도
0 9 * * *매일 오전 9시아침 브리핑
0 9 * * 1매주 월요일 9시주간 리포트
30 22 * * *매일 밤 10시 30분데일리 공지
0 */2 * * *2시간마다 정각파일 동기화
0 9 * * 1-5평일만 9시업무일 전용
⏰ 시간대는 OpenClaw 설정의 timezone을 따릅니다. 한국이면 Asia/Seoul (KST).
4
크론잡 관리
자주 쓰는 명령어 
# 목록 보기
openclaw cron list

# 특정 크론잡 비활성화 (삭제 안 함, 잠시 끄기)
openclaw cron disable <크론잡ID>

# 다시 활성화
openclaw cron enable <크론잡ID>

# 크론잡 삭제
openclaw cron delete <크론잡ID>

# 즉시 한 번 실행 (테스트용)
openclaw cron trigger <크론잡ID>
⚠️ 주의: 같은 시각(같은 시:분)에 여러 크론잡이 실행되면 충돌할 수 있어요. 추가 전에 openclaw cron list로 기존 스케줄을 확인하세요.

Google Drive 연동 (rclone)
팀 파일을 서버와 자동 동기화하는 도구

rclone은 Google Drive, Dropbox 등 클라우드 저장소의 파일을 서버에서 자동으로 올리고 내리는 프로그램입니다. OpenClaw 어시스턴트가 Drive에 있는 팀 파일을 직접 읽고 쓸 수 있게 해줍니다.

왜 필요한가요?
팀원이 Google Drive에 파일을 올리면, 어시스턴트가 자동으로 가져와서 분석·정리합니다. 사람이 "파일 확인해줘" 할 필요 없이 크론잡과 결합하면 완전 자동화됩니다.
현비스는 어떻게 쓰나요?
팀원 인풋 파일 자동 수집, 브랜드별 데이터 동기화, 대시보드 콘텐츠 업데이트 등 하루 6회 이상 자동 동기화를 rclone으로 돌리고 있어요.
1
rclone 설치
Ubuntu 터미널 
curl https://rclone.org/install.sh | sudo bash
2
Google Drive 연결
Ubuntu 터미널 
rclone config

대화형 설정이 시작됩니다. 주요 단계:

  1. n (New remote) 선택
  2. 이름 입력 — 예: gdrive (개인 Drive) 또는 gteam (Shared Drive)
  3. Storage type: drive (Google Drive) 선택
  4. Client ID/Secret: 비워두면 기본값 사용 (대부분 OK)
  5. Scope: 1 (Full access) 선택
  6. 인증: 브라우저가 열리면 Google 계정으로 로그인 + 허용
  7. Shared Drive 사용 시: Configure this as a Shared Drive? → y
💡 WSL2에서는 브라우저가 안 열릴 수 있어요. 그때는 표시되는 URL을 윈도우 브라우저에 직접 복붙해서 인증하면 됩니다.
3
연결 확인
Ubuntu 터미널 
# Drive 루트 폴더 목록 확인
rclone ls gdrive: --max-depth 1

# Shared Drive라면
rclone ls gteam: --max-depth 1

파일 목록이 보이면 연결 성공입니다.

4
파일 가져오기 / 올리기
자주 쓰는 명령어 
# Drive → 로컬 (다운로드)
rclone copy gteam:03_input/ ~/workspace/gdrive-sync/03_input/

# 로컬 → Drive (업로드)
rclone copy ~/workspace/gdrive-sync/result.md gteam:00_ops/
🚨 절대 주의: rclone sync는 사용 금지! sync는 대상 폴더에 없는 파일을 삭제합니다. 항상 rclone copy만 사용하세요. 실수로 sync 쓰면 Drive 파일이 날아갈 수 있어요.
5
크론잡과 연결 (자동 동기화)

크론잡에서 rclone 명령을 실행하면 정기적으로 자동 동기화됩니다. 예를 들어:

크론잡 프롬프트 예시 
Google Drive 03_input 폴더에서 새 파일 가져와서
~/workspace/gdrive-sync/03_input/에 저장해줘.
이미 있는 파일은 건너뛰고.

어시스턴트가 rclone copy 명령을 실행해서 처리합니다.

4-1 invalid_auth — 슬랙 인증 실패

4-2 missing_scope — 슬랙 권한 부족

원인: 슬랙 앱에 Bot Token Scope가 빠져 있음

해결: api.slack.com/apps → 앱 → OAuth & Permissions → Bot Scopes에 아래 항목 모두 확인:

필수 Scopes
app_mentions:read  channels:history  channels:read
chat:write         files:read        files:write
groups:history     groups:read
im:history         im:read           im:write
mpim:history       mpim:read
reactions:read     reactions:write   users:read

Scope 추가 후 반드시 Reinstall to Workspace 클릭!

4-3 Gateway not reachable — 게이트웨이 안 뜸

원인: 프로세스가 죽었거나 포트(18789) 충돌

Ubuntu 터미널 
# 포그라운드로 직접 실행해서 에러 확인
openclaw gateway --port 18789

# 포트 충돌 확인
ss -tlnp | grep 18789

# 기존 프로세스 강제 종료 후 재시작
kill -9 $(ss -tlnp | grep 18789 | awk '{print $6}' | grep -oP '(?<=pid=)\d+')
openclaw gateway restart

설정 순서 (벤이 직접 진행):

  1. github.com/signup → 이메일 ben@gromeet.com으로 계정 생성
    추천 username: gromeet-benvis 또는 benvis-bot

  4. Vercel 로그인 → Account Settings → Connected Accounts → GitHub 연결
  5. gromeet-team org에 멤버로 추가 요청 (현비스가 GitHub API로 처리)
💡 완료 후 벤비스가 어느 레포에서 push해도 Vercel 배포 정상 작동. 현비스(gromeet), 코워크(gromeetmk-pixel), 벤비스(신규) 3계정 모두 커버됨.

🧠
핵심 개념 (비개발자용)
자주 나오는 용어를 쉽게 설명합니다
💬 세션 (Session)
현비스와의 대화 창
슬랙 DM 하나가 1개 세션입니다. 세션이 끝나면(타임아웃 or 리셋) 기억이 초기화됩니다. 그래서 중요한 건 MEMORY.md에 저장해둬야 합니다.
⏰ 크론잡 (Cron Job)
정해진 시간에 자동 실행되는 예약 작업
예: 매시간 드라이브 동기화, 매일 22:30 데일리 공지. "알람 시계"처럼 설정해두면 현비스가 자동으로 실행합니다.
🤖 에이전트 (Agent)
개발 작업을 대신 해주는 AI 코더
현비스가 지시하면 코드를 작성하고 배포합니다. 현재는 OpenAI Codex OAuth 기반 경로를 우선 사용하며, 코딩/파일 수정/배포까지 자동으로 처리합니다.
🧬 서브에이전트 (Sub-agent)
현비스가 다른 AI를 불러 병렬로 일시키는 것
복잡한 작업을 여러 AI가 동시에 처리합니다. 장점: 빠름. 단점: 가끔 먹통이거나 컨텍스트 공유가 안 될 수 있음.
🖥️ tmux 세션
터미널 창을 백그라운드에서 유지하는 기술
에이전트가 tmux 안에서 작업합니다. 터미널을 닫아도 작업이 계속 돌아가요. 여러 에이전트를 동시에 실행할 때 필수.
🚪 게이트웨이 (Gateway)
슬랙과 현비스를 연결하는 다리
이게 꺼지면 현비스가 슬랙 메시지를 못 받습니다. openclaw gateway status로 확인. 시스템 서비스로 자동 시작 설정 가능.
🧠 메모리 시스템
AI의 기억 관리 체계
memory/날짜.md = 오늘 일기, MEMORY.md = 장기 기억, LEARNINGS.md = 실수에서 배운 교훈. 세션이 끝나도 이 파일들은 유지됩니다.
🛠️ 스킬 (Skill)
현비스의 재사용 가능한 전문 기능
input-organizer(인풋 정리), sprint-manager(스프린트 관리) 등. .skills/ 폴더에 레시피처럼 저장되어 있습니다.
📊 data.js / generate-data.py
워크스페이스 가이드 사이트의 데이터
매시간 Google Drive에서 폴더 구조를 읽어서 data.js를 자동 생성합니다. generate-data.py가 이 작업을 수행.
🌐 Vercel
웹사이트를 인터넷에 올려주는 서비스
대시보드, 워크스페이스 가이드 등이 Vercel에 배포되어 있습니다. GitHub 연동 또는 CLI로 배포합니다.
☁️ rclone
Google Drive 파일을 서버에서 자동 동기화
파일 올리기/내리기/목록 조회를 CLI로 자동화합니다. 크론잡과 함께 매시간 Drive 데이터를 서버에 동기화할 때 사용.
🪙 토큰 (Token)
AI가 읽고 쓰는 글자의 단위
한글 1글자 ≈ 2~3 토큰. AI에게 보내는 메시지 + 파일 내용 + AI의 답변 전부 토큰으로 계산됩니다. 토큰을 많이 쓸수록 비용이 올라가요.
📦 컨텍스트 윈도우 (Context Window)
AI가 한 번에 기억할 수 있는 범위
책상 크기라고 생각하세요. 대화가 길어지면 책상이 꽉 차서 오래된 내용을 치워야 합니다. 이때 "컴팩션"이 일어나요.
🗜️ 컴팩션 (Compaction)
긴 대화를 요약해서 공간을 확보하는 것
컨텍스트 윈도우가 가득 차면 OpenClaw이 이전 대화를 자동 요약합니다. 핵심만 남기고 세부 내용은 사라져요. 중요한 결정은 반드시 MEMORY.md에 저장해야 하는 이유입니다.
📥 자동주입 (Auto-injection)
세션 시작 시 파일이 자동으로 AI에게 전달되는 것
AGENTS.md, SOUL.md 등 8개 파일이 매 세션마다 자동으로 "책상 위에 올라갑니다". AI가 안 읽을 수가 없어요. 반면 RULES.md 같은 파일은 직접 "서랍에서 꺼내 읽으라"고 지시해야 합니다.
💓 하트비트 (Heartbeat)
30분마다 "이상 없나?" 자동 점검
HEARTBEAT.md 파일에 점검 항목을 정의해두면 OpenClaw이 30분마다 체크합니다. 이상 없으면 조용히, 문제 있을 때만 DM으로 알려줍니다.
🧪 모델 (Model)
AI의 두뇌 종류
GPT-5.4 = 주력 엔진 (코딩, 구조화, 정확도). GPT-5.2 = 일반 업무용 (빠른 응답, 가벼운 잡무). Opus 4.6은 필요할 때만 쓰는 선택 fallback입니다.
📝 프롬프트 (Prompt)
AI에게 보내는 지시문
여러분이 슬랙에 쓰는 메시지도 프롬프트입니다. AGENTS.md, SOUL.md 같은 파일도 "시스템 프롬프트"로 매번 AI에게 전달됩니다. 잘 쓸수록 AI 결과가 좋아져요.
📂 워크스페이스 (Workspace)
AI 비서의 작업 폴더
~/.openclaw/workspace/ 안에 모든 설정 파일, 메모리, 스크립트가 들어있습니다. AI 비서의 "사무실"이라고 보면 됩니다.
🔌 API (Application Programming Interface)
프로그램끼리 대화하는 창구
슬랙 API로 메시지 읽기/보내기, Google Calendar API로 일정 조회, Notion API로 DB 조작. 현비스가 외부 서비스를 "손"처럼 쓸 수 있게 해주는 통로입니다.
🔐 OAuth (인증 프로토콜)
비밀번호 없이 안전하게 로그인하는 방식
"Google로 로그인" 버튼과 같은 원리. 현재는 OpenAI Codex OAuth로 GPT를 연결하는 데 주로 사용합니다. 토큰이 만료되면 재인증이 필요해요.
📱 노드 (Node)
OpenClaw에 연결된 기기
스마트폰, 다른 PC 등을 OpenClaw에 페어링하면 카메라 촬영, 화면 캡처, 위치 조회 등을 원격으로 할 수 있습니다.
🌿 Git / GitHub
코드 변경 이력을 관리하는 시스템
파일 수정 내역을 저장하는 "타임머신". GitHub에 올리면(push) Vercel이 자동으로 웹사이트를 업데이트합니다. 대시보드가 업데이트되는 원리예요.
🔄 채널 라우팅 (Channel Routing)
여러 메신저를 하나의 AI로 연결하는 것
슬랙, 텔레그램, 시그널 등 여러 채널에서 온 메시지를 하나의 AI 비서가 처리합니다. 어디서 말을 걸어도 같은 비서가 응답해요.

세션이란?

세션은 대화 창 하나입니다. 카카오톡 채팅방 하나를 떠올리면 됩니다. 세션이 시작되면 현비스가 "부팅"을 합니다.

세션 시작 시
자동주입 파일 8개를 읽고 → 규칙/성격/기억을 불러옴 → "부팅 시퀀스" 완료 → 대화 준비 완료
세션 종료 조건
타임아웃: 일정 시간 대화 없으면 자동 종료
/reset: 수동으로 세션 초기화
컴팩션
대화가 길어지면 AI가 이전 내용을 요약해서 공간을 확보합니다. 책의 앞부분을 요약본으로 바꾸는 것과 같습니다.
하트비트
30분마다 "이상 없나?" 전체 시스템을 자동 체크합니다. 문제 발견 시 슬랙으로 알림을 보냅니다.
자동주입 vs 수동읽기

현비스가 참고하는 문서는 두 종류입니다. 비유하면:

📎 자동주입 = 책상 위 서류
세션 시작 시 무조건 읽습니다. 항상 눈앞에 있으니 규칙을 잘 지킵니다.
✋ 수동읽기 = 서랍 속 서류
필요할 때만 꺼내 봅니다. 까먹고 안 볼 수도 있어서, 중요한 건 자동주입에 넣어야 합니다.

AI 모델이란?

AI 모델은 두뇌의 종류입니다. 현재 그로밋팀 기준은 GPT-first이고, 필요할 때만 다른 모델을 보조로 붙입니다.

🟣 GPT-5.4 (주력)
정확도 · 개발 · 운영
복잡한 코딩, 분석, 의사결정에 사용하는 기본 모델입니다. 현재 현비스의 핵심 운영 경로입니다.
🔵 GPT-5.2 (일반 업무)
빠름 · 가벼움
짧은 질문, 일반 대화, 빠른 정리에 적합합니다. 너무 무거운 작업까지 맡기지 않는 것이 좋습니다.
상황사용 모델이유
일반 대화GPT-5.2빠른 응답이 중요
코딩/개발GPT-5.4정확성과 일관성이 중요
서브에이전트GPT-5.4메인과 같은 기준 유지
크론잡GPT-5.4운영 안정성 우선
💡 Claude/Opus는 완전히 버린 게 아니라 선택 fallback입니다. 다만 운영 표준 문서와 기본 세팅은 GPT 기준으로 맞추는 게 현재 팀 정책입니다.
크론잡 vs 하트비트

둘 다 자동 실행이지만 목적이 다릅니다.

⏰ 크론잡
정해진 시간에 특정 작업을 실행합니다.
예: 매일 22:30 데일리 공지, 매시간 드라이브 동기화, 매일 09:00 인풋 체크

비유: 알람 시계 — "이 시간에 이것을 해라"
💓 하트비트
30분마다 전체 상태를 점검합니다.
예: 게이트웨이 살아있나? 크론잡 밀린 거 없나? 에러 난 거 없나?

비유: 순찰 경비 — "30분마다 이상 없나 돌아보기"
📂
워크스페이스 문서 가이드
각 파일이 뭐하는 건지, 비개발자용 설명

현비스 워크스페이스에는 여러 .md 파일이 있습니다. 각 파일은 현비스에게 특정 역할을 부여합니다. 아래에서 하나씩 알아봅시다.

📌 🔄 자동주입 = 세션 시작 시 자동으로 읽히는 파일  |  ✋ 수동읽기 = 필요할 때만 읽는 파일
🤖 AGENTS.md 🔄 자동주입
운영 매뉴얼
현비스의 부팅 순서, 메모리 정책, 팀 운영 방법이 적혀있습니다. 회사의 "사원 핸드북"과 같습니다.
⛔ 이 파일이 없으면? 현비스가 부팅 순서를 모르고, 팀 운영 규칙을 어길 수 있습니다.
💜 SOUL.md 🔄 자동주입
성격과 말투
간결하게 말하기, 전문 용어에 쉬운 설명 병기, 채널별 톤 조절 등 현비스의 "성격"을 정의합니다.
⛔ 이 파일이 없으면? 현비스가 딱딱하거나 장황하게 말할 수 있습니다.
👤 USER.md 🔄 자동주입
사용자(그로밋) 정보
그로밋의 브랜드 목록, 팀 구성, 선호 사항 등 사용자 맥락을 담고 있습니다. "고객 카드"와 같습니다.
⛔ 이 파일이 없으면? 현비스가 그로밋의 브랜드나 팀 구조를 모르고 엉뚱한 답을 할 수 있습니다.
🪪 IDENTITY.md 🔄 자동주입
현비스 정체성
현비스의 이름, 이모지(🐾), 아바타 설정 등 "나는 누구인가"를 정의합니다.
⛔ 이 파일이 없으면? 현비스가 자기 이름이나 이모지를 모르고, 다른 AI처럼 행동할 수 있습니다.
🛠️ TOOLS.md 🔄 자동주입
도구 사용 메모
카메라, SSH, TTS 등 연동된 도구의 사용법과 설정을 기록합니다. "장비 매뉴얼"과 같습니다.
⛔ 이 파일이 없으면? 도구 연동 시 설정을 매번 물어보거나 잘못된 설정으로 실행할 수 있습니다.
💓 HEARTBEAT.md 🔄 자동주입
30분 주기 체크 규칙
하트비트 때 무엇을 체크할지, 이상 발견 시 어떻게 알릴지를 정의합니다. "순찰 체크리스트"입니다.
⛔ 이 파일이 없으면? 시스템 이상을 감지하지 못하고, 장애가 방치될 수 있습니다.
🚀 BOOTSTRAP.md 🔄 자동주입
부팅 순서 정의
세션 시작 시 어떤 파일을 어떤 순서로 읽을지 지시합니다. 컴퓨터의 "BIOS 설정"과 같습니다.
⛔ 이 파일이 없으면? 부팅 순서가 엉키면 중요한 규칙을 빠뜨릴 수 있습니다.
🧠 MEMORY.md 🔄 자동주입
장기 기억
중요한 결정, 프로젝트 히스토리, 핵심 맥락을 저장합니다. 세션이 바뀌어도 유지되는 "업무 일지"입니다.
⛔ 이 파일이 없으면? 과거 결정이나 프로젝트 맥락을 매번 새로 설명해야 합니다.
🚫 RULES.md ✋ 수동읽기
절대 규칙
삭제 금지, rclone copy only, 특정 파일 건드리지 말 것 등 "절대로 어기면 안 되는 규칙"입니다.
⛔ 이 파일을 안 읽으면? 금지된 행동(파일 삭제, 위험한 명령)을 실행할 위험이 있습니다.
📏 gromit-rules.md ✋ 수동읽기
채널/팀/브랜드 규칙 (SSOT)
채널 라우팅, 팀 구조, 브랜드별 규칙의 단일 진실 소스(SSOT)입니다. 변경 시 이 파일만 수정하면 됩니다.
⛔ 이 파일을 안 읽으면? 채널 라우팅을 틀리거나, 브랜드 규칙을 혼동할 수 있습니다.
📝 LEARNINGS.md ✋ 수동읽기
과거 실수 기록
과거에 했던 실수와 그로부터 배운 교훈을 기록합니다. 같은 실수를 반복하지 않기 위한 "오답 노트"입니다.
⛔ 이 파일을 안 읽으면? 같은 실수를 또 할 수 있습니다 (예: 잘못된 명령어 재사용).
📅 memory/날짜.md ✋ 수동읽기
일일 기록
오늘 있었던 일, 처리한 작업, 발견한 이슈를 기록합니다. 매일 새 파일이 생기는 "업무 일기"입니다.
⛔ 이 파일을 안 읽으면? 오늘 있었던 일의 맥락을 모른 채 대화하게 됩니다.
✅ 자동주입 파일 8개는 세션마다 자동으로 읽히니 신경 쓸 필요 없습니다. 수동읽기 파일은 현비스가 필요할 때 스스로 읽지만, 중요한 규칙 변경 시에는 "RULES.md 다시 읽어봐"라고 말해주면 확실합니다.
